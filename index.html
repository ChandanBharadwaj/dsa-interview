Zero to Interview-Ready in 14 Weeks

A complete DSA runbook for an experienced Java engineer starting from scratch. Concepts → animations → Java templates → when-to-use-what → real systems → practice.

How to use this runbook. Go in order — every topic builds on the ones before it. For each topic: read the page top to bottom, play the animations until the mechanics feel obvious, type out (don't copy) the pattern template, then solve the curated problems easy→hard, only opening a solution after 25–30 minutes of honest effort. Re-solve failed problems 2 days later. Tick the checkbox when you can solve the worked problems cold. Your progress is saved in this browser.

The journey at a glance

Each phase unlocks the next: array patterns make sorting natural, recursion makes trees natural, trees make graphs natural, and graphs + recursion make dynamic programming learnable.

Final-week ritual. Re-do one worked problem from every topic without looking. Anything shaky → that page's pitfalls section. Then do 2–3 full mock interviews out loud with a timer: restate the problem, state brute force + its complexity, improve it, code, test with an example, state final complexity. That 6-step out-loud loop is the actual interview skill.
